# Web App Spec тАФ `apps/web`

**Stack:** Astro + Tailwind CSS + minimal React islands.
**Deploy:** Cloudflare Pages.
**Domain:** `zephyr.build` (and `www.zephyr.build` redirect).

---

## Pages

### `/` тАФ Landing

**Sections, top to bottom:**

1. **Nav bar** тАФ left: Zephyr wordmark. Right: Leaderboard, Docs, GitHub, "Powered by Weaverse" link.
2. **Hero**
   - Headline: **Shopify built the rails.**
   - Sub-headline: **Zephyr tells you if your store is on them.**
   - Scan input (large, centered): `[ https://yourstore.com ]  [ Scan now тЖТ ]`
   - Subtext: "Free. No signup. Built by [Weaverse](https://weaverse.io)."
3. **"What we check"** тАФ 5 category cards in a row (Discoverability, Content, Commerce, Product Data, Checkout). Each card lists 2-3 checks + a tooltip.
4. **Live leaderboard preview** тАФ top 5 scores, "See full leaderboard тЖТ"
5. **How it works** тАФ 3 steps with icons:
   - Paste your URL
   - We run 15 checks across 5 commerce-readiness categories
   - Get a score + fix-it action list
6. **For developers** тАФ short pitch for embedding the badge + open-source link
7. **Footer** тАФ links, MIT, GitHub, Weaverse

### `/scan/[domain]` тАФ Result page

Fetches `GET /scan?url={domain}` server-side in Astro, renders SSR.

**Sections:**

1. **Result header** тАФ big domain name, scan timestamp, "Re-scan" button (links with `?fresh=true`)
2. **Big score** тАФ large circular gauge, overall number + grade letter. Color: green (A/B), yellow (C), red (D/F).
3. **Limited-coverage banner** (conditional, when `meta.limitedCoverage === true`):
   > тЪая╕П Scanner is in early development. This score reflects {checksCovered} of {checksTotal} planned checks. Full coverage coming soon.
4. **Category breakdown** тАФ 5 horizontal bars with sub-scores
5. **Check list** тАФ grouped by category. Passed checks collapsed by default. Failed checks expanded showing `detail` + `fixHint` + `evidence` (the evidence as a small `<details><summary>Raw data</summary>` block).
6. **Share row**
   - Twitter share button (pre-filled tweet)
   - Copy link button
   - Embed badge code (textarea + copy button)
7. **CTA card** тАФ "Built on Hydrogen? Get these issues fixed automatically with [Weaverse тЖТ]"

### `/leaderboard`

Fetches `GET /leaderboard`. Sortable table:
- Rank, Domain (linked to `/scan/{domain}`), Score, Grade, Scanned at
- Filter dropdown: All / Discoverability / Commerce / etc.
- Period dropdown: 7d / 30d / all

### `/about`

Static. Explains:
- What's a check (link to `CHECKS.md`)
- Who built it (Weaverse)
- Protocols covered (UCP, ACP, MPP, x402, WebMCP, MCP)
- How to contribute (link to `CONTRIBUTING.md`)
- License (MIT)

### `/docs` (optional v1.1)

Redirect to GitHub docs folder.

## React Islands

Astro is static by default. Add `client:load` only where needed:

| Component | Page | Why |
|---|---|---|
| `<ScanInput />` | `/` | Live URL validation + form submit |
| `<ShareButtons />` | `/scan/[domain]` | Clipboard API, dropdown state |
| `<LeaderboardFilter />` | `/leaderboard` | Filter state |
| `<CategoryGauge />` | `/scan/[domain]` | Animated counter (optional) |

Everything else is pure server-rendered HTML.

## SEO

- `<title>` per page
- OG image: rendered with the score badge for `/scan/[domain]` (cached in R2)
- `sitemap.xml` generated by Astro for static pages + dynamic scan slugs (top 100 from leaderboard)
- Canonical URL for `/scan/{domain}` strips query params except `?fresh`

## Analytics

- Cloudflare Web Analytics (privacy-first, no cookies)
- Track: `/`, `/scan/*` (with domain as event prop), `/leaderboard`, scan submissions
